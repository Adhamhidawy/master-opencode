    3. Create `src/components/interactive/quiz-flow.tsx` as a `"use client"` component:
       - Import `quizQuestions` from `@/data/quiz`, `QuizQuestion` from `@/types/quiz`, `cn` from `@/lib/utils`

       - Define a discriminated union state type and useReducer:
         ```typescript
         type QuizState = {
           phase: "question" | "result";
           currentIndex: number;
           answers: (number | null)[];
           selectedAnswer: number | null;
           isAnswered: boolean;
           isAnimating: boolean;
         };
         ```
         Initial state: `{ phase: "question", currentIndex: 0, answers: Array(10).fill(null), selectedAnswer: null, isAnswered: false, isAnimating: false }`

       - Actions: `{ type: "SELECT_ANSWER"; index: number }`, `{ type: "NEXT_QUESTION" }`, `{ type: "SHOW_RESULTS" }`, `{ type: "RESET" }`
       - Reducer logic:
         - SELECT_ANSWER: set `selectedAnswer` and mark `isAnswered: true`, store answer in `answers[currentIndex]`
         - NEXT_QUESTION: if `currentIndex < 9`, set `isAnimating: true`, then after CSS transition, increment `currentIndex`, reset `selectedAnswer: null`, `isAnswered: false`, `isAnimating: false`. If last question, transition to `phase: "result"`
         - SHOW_RESULTS: set `phase: "result"`
         - RESET: return initial state

       - Question view rendering:
         - Progress bar at top: "Question {N+1} of 10" with a thin progress bar (width = (N+1)/10 * 100%) using `bg-accent` color
         - Question card: `bg-bg-2 border border-border rounded-lg p-8`
           - Question text: `text-[1.05rem] font-bold leading-relaxed`
           - 4 option buttons in vertical stack (`flex flex-col gap-2`):
             - Default: `border border-border rounded-sm px-4 py-3.5 text-[.88rem] text-text-2 text-left cursor-pointer hover:border-accent hover:text-text transition-all w-full bg-transparent`
             - When answered & correct: `border-green bg-green/10 text-green-2 pointer-events-none`
             - When answered & wrong selected: `border-red bg-red/10 text-red pointer-events-none`
             - When answered & not selected (not correct): `pointer-events-none opacity-60`
           - After answering, show feedback text below options: "Correct!" (green) or "Incorrect. The answer is: {correctOption}" (red)

         - "Next →" button: appears only when `isAnswered` is true. Styled like CTA buttons: `bg-accent text-white rounded-lg px-6 py-2.5 font-semibold hover:bg-accent-2`. On click: dispatch NEXT_QUESTION (or SHOW_RESULTS if last).
         - Animation: wrap the question content in a div with transition classes. Use `transition-all duration-300`. When `isAnimating` is true, apply `opacity-0 translate-x-4`. When not animating, apply `opacity-100 translate-x-0`. Use `useEffect` + `setTimeout(300)` to handle animation timing before dispatching the actual state change.

       - Result view rendering:
         - Score display: large text "Score: {correct}/10"
         - Pass/fail: if score >= 7: `text-green-2` with "Passed! 🎉", else `text-red` with "Keep practicing!"
         - Retry button: `bg-accent text-white rounded-lg px-8 py-3 font-semibold` with onClick dispatch RESET
         - Back to home link: `<Link href="/">Back to Home</Link>`
